> 用户授权：自主完成全部流程，不中途汇报，高频 git 提交，做到"完整、有趣、无 bug"。
> 工作节奏：每任务先写 `plans/NN_*.md`（目标 + 效果预期）→ 实现 → headless 验证零报错 → 对照目标 → commit。
> 平台硬约束：纯安卓手机游戏，横屏，触屏操作，最终交付 APK（arm64-v8a，已签名）。

## 验收红线
- 每次 headless 运行 `--quit-after 12` 必须零 SCRIPT ERROR / 零 Animation not found；`is_inside_tree` 良性警告不计。
- 所有新信号/节点在场景与脚本中一致；不引入未定义引用。
- 最终 APK 必须 `apksigner verify` 通过、含 arm64 原生库、manifest 横屏。
- 不破坏已有系统（成长/进食/存档/食物链/触屏）。

- 不做联网多人（Phase4，沙箱无服务器，且用户要"无 bug"；留作后续）。
- 不重写架构、不推翻现有代码（增量开发原则）。
